这个一个 **基于资料的学习教练**的skill，不是普通总结工具
把资料和学习时间给到他，他会帮你主动设计知识学习的路径

它主要解决这类问题：

1. **看不懂陌生资料**
   
   比如 PDF、PPT、官网、课程文档、产品介绍、客户资料、行业报告等。它会先读资料，再带你拆解，不会直接凭感觉讲。

2. **资料很多，不知道从哪里学**
   
   它会根据你的用途和时间，帮你决定先学什么、跳过什么、后面补什么。

3. **听懂了但不会用**
   
   它不只总结内容，还会问你理解问题、纠偏，并引导你把知识用到实际场景里，比如改稿、汇报、客户拜访、销售话术、学习笔记。

4. **担心 AI 瞎发挥**
   
   它会把内容分成：
   
   ```text
   资料明确写到
   基于资料的解释
   基于资料的推断
   资料未说明，需要向资料提供者/业务方/作者确认
   ```

5. **学习过程太快、太散**
   
   它会主动控节奏：一次只讲一个点，讲完停下来确认理解，你可以随时追问，理解后回复“继续”。

**它是如何运行的**

当你发来资料，或者说“用 source-grounded-learning-coach 帮我学这个资料”，它会按这个流程走：

```text
读取资料
-> 确认你的使用场景和学习时间
-> 给出本次学习计划和预期成果
-> 一小节一小节讲
-> 检查理解、回答追问、纠偏
-> 根据进度生成学习成果
```

开场时它会先问两个关键问题：

```text
你这次主要想用这份资料做什么？
大概有多久学习？
```

然后根据你的回答安排计划，比如：

```text
本次能学到什么
本次不能覆盖什么
学完能做出什么判断或产物
后续可以继续补什么
```

学习中，每一小节会尽量保持这个结构：

```text
资料明确写到
基于资料的解释
例子或应用
理解检查
```

如果你回答得好，它会具体指出你抓对了哪里；如果你理解偏了，它会先鼓励，再指出问题，并给更准确版本。

适合用它处理：课程学习、短视频/写作方法论、产品资料、客户研究、官网学习、领导答疑准备、销售/市场材料学习。

不适合用它做的事：不给资料就硬编结论，或者跳过学习过程直接产出一篇看似完整但不一定扎实的稿子。
