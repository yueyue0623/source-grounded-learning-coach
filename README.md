# Source-Grounded Learning Coach — 产品说明文档

> **中文名**：基于资料的学习教练  
> **定位**：以资料为锚点的结构化学习教练，帮助用户把陌生材料转化为可用的理解和产出物。

---

## 一、产品概述

Source-Grounded Learning Coach 是一个 **学习导向** 的 Skill，不是被动总结工具，也不是快速生成器。它的核心任务是：**带你一步一步学透一份资料，并在这个过程中培养你对资料的准确理解能力。**

它解决的核心矛盾是——AI 帮你"理解"资料时经常出现的两个问题：

1. **AI 替你总结，你失去了学习过程。**
2. **AI 自由发挥，你分不清哪些是资料说的、哪些是 AI 编的。**

这个 Skill 用一套严格的规则解决了这两个问题。

---

## 二、解决什么问题

| 痛点 | 本 Skill 的处理方式 |
|------|-------------------|
| **看不懂陌生资料**（PDF、PPT、官网、产品文档、行业报告等） | 先读取资料，再带你拆解，不凭感觉讲 |
| **资料很多，不知道从哪学** | 根据你的用途和时间，帮你决定先学什么、跳过什么、后续补什么 |
| **听懂了但不会用** | 不只总结，还提问、纠偏，把知识引导到实际场景（改稿、汇报、客户拜访、销售话术等） |
| **担心 AI 瞎发挥** | 每个结论都标注来源层级：资料写到 / 基于资料解释 / 基于资料推断 / 需确认 |
| **学习过程太快、太散** | 主动控节奏：一次只讲一个点，讲完停下确认理解 |

## 三、核心设计原则

### 3.1 四层证据标注

所有内容必须明确标注信息来源层级：

```
资料明确写到        → 原文直接陈述的事实
基于资料的解释      → 对原文的通俗化翻译
基于资料的推断      → 超出原文但合理的延伸
资料未说明，需确认  → 资料未覆盖，需要向提供方/业务方核实
```

这一机制确保用户始终知道**哪些是确定的、哪些是推演的、哪些还存疑**。

### 3.2 小节教学 + 节奏控制

- **每次只讲一个知识点**，讲完停下。
- 用户确认理解后（回复"继续"或提问），才进入下一节。
- 全程显示进度（如 `已完成 3/8`）。
- 不同时长匹配不同深度：

| 可用时间 | 覆盖范围 |
|---------|---------|
| 10–15 分钟 | 核心理念 + 一个例子 + 一个收获 |
| 30–45 分钟 | 核心框架 + 2–3 个例子 + 基础应用 |
| 60–90 分钟 | 完整学习地图 + 练习 + 纠偏 + 产出初稿 |
| 多次会话 | 分阶段计划，每阶段有里程碑和产出物 |

### 3.3 客观反馈机制

对用户的理解不敷衍、不打击。用结构化方式反馈：

```
准确的地方        → 明确指出答对了什么
需要修正的地方    → 指出偏差在哪儿
更准确版本        → 给出正确理解
当前水平/下一步    → 帮用户看到成长路径
```

### 3.4 实用导向排序

**不按 PPT 页码死讲**，而是围绕用户的真实使用场景重新组织内容：

- **学习/培训类材料**：核心理念 → 关键区分 → 方法框架 → 正反例 → 常见错误 → 练习
- **公司/产品/客户类材料**：产品定位 → 客户场景 → 痛点 → 能力 → 案例 → 边界 → 竞品

## 五、适用场景与产出物对照

| 使用场景 | 典型产出物 |
|---------|-----------|
| **自学理解**（课程、方法论、行业资料） | 学习笔记、概念图谱、术语表 |
| **改稿/创作**（内容方法、写作指南） | 改写检查清单、练习提示、改前/改后示例 |
| **准备汇报**（产品资料、市场报告） | 简洁理解摘要 + 高价值问题 + 工作优先级 |
| **给领导答疑** | 一页纸答疑提纲（初步理解 + 待确认问题） |
| **调研客户** | 客户背景简析、痛点匹配、未解问题、拜访准备 |
| **准备客户拜访** | 销售话术、挖掘问题、异议处理、竞品对比 |
| **做市场内容** | 客户细分角度、痛点选题、案例包装 |
| **入职自学**（产品文档、入职材料） | 概念图谱、逐节学习、自测题 |

## 六、适合与不适合

### ✅ 适合用它处理

- 课程学习、短视频/写作方法论
- 产品资料、客户研究、官网学习
- 行业报告、竞品分析
- 销售/市场材料理解
- 领导答疑准备
- 入职材料自学

### ❌ 不适合用它处理

- **不给资料就硬编结论**：必须有资料来源
- **跳过学习过程直接出成品**：这是学习教练，不是代写工具（可以产出学习笔记等学习产物，但不是跳过理解直接给你一篇完整稿子）
- **快速总结**：如果你只想看摘要、不想被提问和纠偏，用普通对话即可

## 七、如何安装

### 安装方法

直接把 Skill 文件（SKILL.md）拖入 Agent 的对话框，让 Agent 帮你安装。注意该 Agent 需要支持 Skill 功能。

### 验证安装

开启新对话，直接提供资料并说明学习需求即可触发。如果 Skill 正确加载，它会以"我会先读资料，再带你一小节一小节学……"开场。

### 安装后无需额外配置

Skill 安装完成后即刻生效，无需重启或手动激活。Agent 会根据对话内容自动判断是否需要加载此 Skill。

## 八、如何使用

### 触发方式

在对话中提供资料并说明需求即可，例如：

- "帮我看一下这份产品PPT，我要准备明天的客户拜访"
- "用 source-grounded-learning-coach 帮我学这份行业报告"
- "我有一份竞品资料，想做个对比分析"

提供的资料形式可以是：
- 本地文件（PDF、PPT、Word、Markdown 等）
- 网页 URL
- 截图
- 直接粘贴的文本

### 你只需要回答两个问题

Skill 启动后会问你：

1. **你这次主要想用这份资料做什么？**（自学 / 汇报 / 客户拜访 / 市场内容……）
2. **大概有多久学习？**（15分钟 / 45分钟 / 不限……）

然后它会根据你的回答定制学习计划和节奏。

### 学习过程中的操作

- 理解了一个点 → 回复 **"继续"** 进入下一节
- 有疑问 → 随时提问，Skill 会先解答再继续
- 觉得太快 → 说"慢一点"，Skill 会缩小范围、放慢节奏
- 觉得太慢 → 说"快一点"，Skill 会加速但不会跳过检查

### 产出物

学习结束后，Skill 会根据你的场景生成对应的产出物（学习笔记、话术、检查清单、答疑提纲等）。这些产出物**基于你与 Skill 的互动中确认过的理解**，不是凭空生成的。

---

## 九、关键行为准则（给 Skill 使用者）

使用这个 Skill 时，它会严格遵循以下规则，了解这些规则有助于你更好地配合：

1. **不一次倒出全部内容**：它会分段讲，你需要主动回应才能推进。
2. **不把推测当事实**：每条信息都会标注来源层级，你永远知道什么是确定、什么是推断。
3. **不泛泛夸奖**：它会具体指出你理解对或错的地方，目的是帮你真正掌握。
4. **资料不够时不硬编**：如果资料缺失，它会先基于现有内容继续教学，并在相关小节后标注"资料不足提醒"。
5. **不按页码死讲**：内容会围绕你的使用场景重新组织，而不是按 PPT 顺序走。

## 十、典型对话示例

```
用户：帮我看一下这份云分账产品PPT，我要准备下周的客户拜访

Skill：我会先读资料，再带你一小节一小节学；资料事实、基于资料的解释、
资料没说明的部分会分开讲。我会主动把控学习节奏：每次只讲一个点，讲完会
停下来确认理解。你可以随时提问，理解了回复"继续"即可。
你这次主要想用这份资料做什么？大概有多久学习？

用户：准备客户拜访，大概30分钟

Skill：[给出学习计划]
本次能学到什么：产品核心能力、目标客户痛点、一个典型客户场景
本次不能覆盖什么：完整竞品对比、所有客户案例、定价细节
学完能做出什么判断或产物：客户拜访话术草稿 + 关键挖掘问题清单
这个节奏可以吗？

用户：可以

Skill：[开始第一节教学]
### 已完成 0/7 → 1/7

**资料明确写到**：云分账是基于直连银行账户系统的SaaS化分账服务。
**基于资料的解释**：简单说，它不是把钱打到一个账户再人工分，而是钱进来后
系统自动按规则分到各方的银行账户里。
**对你的场景有什么用**：拜访时如果客户问"这和普通转账有什么区别"，
你可以直接讲"自动化分账 + 银行级合规"这个点。
**边界说明**：资料讲的是收款后分账这个环节，上下游的营销、催收不在产品
范围内。

请你用自己的话说一下：云分账和普通转账汇款的核心区别是什么？
```
